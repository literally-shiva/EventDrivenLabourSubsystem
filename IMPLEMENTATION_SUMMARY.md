## Визуализируемые метрики

1. **Collision Count** (красный) — количество коллизий в BIM-модели
2. **Changes Count** (оранжевый) — количество изменений технического задания
3. **Rework Count** (фиолетовый) — количество возвратов на доработку
4. **Approval Delays** (синий) — задержки получения технических условий (дни)
5. **Documentation Versions** (зелёный) — количество документов на согласовании
6. **Approval Count** (розовый) — количество согласований

## Характерные сигнатуры событий

### CollisionDetected
```
График коллизий: ────┐
                     │+3-7
                     └─────
```
Резкий скачок коллизий на 3-7 единиц.

### DesignRequirementChanged
```
График изменений ТЗ: ───┐
                        │+2-5
                        └────
График документов:   ──┐
                       │+1-2
                       └─────
```
Скачок изменений + дополнительные документы.

### ApprovalDelayed
```
График документов:   ────┐
                         │+3-6
                         └────
График задержек:     ──────┐
                           │5-12 дней
                           └────
```
Резкое накопление документов + значительная задержка.

### DocumentationReturned / ExpertReviewFailed
```
График возвратов:    ────┐
                         │+2-4
                         └────
График документов:   ───┐
                        │+2-4
                        └─────
```
Скачок возвратов + больше версий документов.

### ResourceShortage
```
График работников:   ────┐
                         │-2-3
                         └────
```
Временное снижение количества работников.

## Как тестировать

### Шаг 1: Запуск сервисов

```bash
# Terminal 1: CoreServer
cd solution/CoreServer/src/CoreServer.API
dotnet run

# Terminal 2: DigitalTwin
cd solution/DigitalTwin/src/DigitalTwin.API
dotnet run

# Terminal 3: MLService
cd solution/MLService
python app.py

# Terminal 4: ProjectClient
cd solution/ProjectClient
npm start
```

### Шаг 2: Создание проекта и запуск симуляции

1. Откройте http://localhost:4200
2. Создайте проект с 2-3 работами
3. Сохраните проект
4. Запустите симуляцию (кнопка "Start simulation")
5. **Подождите 1-2 минуты** для накопления данных

### Шаг 3: Просмотр графика

1. Переключитесь на вкладку **Metrics**
2. Проверьте консоль браузера (F12):
   ```
   Loaded metrics: X records
   Sample metric: {...}
   Max values: {...}
   ```
3. На графике должны быть видны:
   - 6 разноцветных линий
   - Цветные точки событий внизу
   - Даты и время по горизонтали
   - Значения по вертикали

### Шаг 4: Наблюдение за паттернами

**Начало проекта (0-20%):**
- Низкие значения метрик (1-3)
- Редкие события
- Возможны: ResourceShortage, DesignRequirementChanged

**Разработка (20-50%):**
- Рост коллизий до 5-10
- Частые события CollisionDetected
- Видимые скачки на графике

**Интеграция (50-80%):**
- Много документов (6-10)
- Задержки согласований
- События: ApprovalDelayed, DocumentationReturned

**Завершение (80-100%):**
- Возвраты на доработку (3-7)
- Финальные согласования
- События: ExpertReviewFailed, DocumentationReturned

## Сравнение: До и После

### До улучшений
```
График коллизий:  ~~~~~~~~~
                  (хаотичный шум)
```
- Невозможно различить события
- Нет накопления
- Случайные значения каждый тик

### После улучшений
```
График коллизий:  ──┐    ┌──┐
                    └────┘  └───
                  (структурированные паттерны)
```
- Чёткие скачки при событиях
- Накопительный рост
- Реалистичные тренды

## Известные ограничения

1. **Отсутствующие метрики** (нет в модели данных):
   - Количество замечаний экспертизы (частично покрывается ReworkCount)
   - Количество несогласованных решений
   - Счётчик изменений BIM-модели (есть только объём)
   - Количество нормативных несоответствий

2. **Нет автообновления графика** — нужно переключиться на другую вкладку и вернуться

3. **Нет интерактивности** — нет tooltip, zoom, pan

4. **Состояние не сохраняется** — при перезапуске DigitalTwin накопительные метрики сбрасываются
